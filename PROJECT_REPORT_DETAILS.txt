PROBLEM DEFINITION
================================================================================
Travelling Salesman Problem (TSP):
- Given N cities on a 2D plane with coordinates (x, y)
- Find the shortest route that visits each city exactly once
- Return to the starting city
- Minimize total travel distance

Complexity: NP-Hard problem
Total Possible Routes: (n-1)!/2 for n cities

Distance Calculation:
Euclidean Distance Formula: d = √((x₂ - x₁)² + (y₂ - y₁)²)


ALGORITHMS IMPLEMENTED
================================================================================

1. BRUTE FORCE ALGORITHM
   ----------------------
   Approach: Exhaustive search - evaluates all possible permutations
   
   Time Complexity: O(n!)
   Space Complexity: O(n)
   
   Advantages:
   - Guarantees optimal solution (shortest possible route)
   - Simple to understand and implement
   
   Disadvantages:
   - Extremely slow for large inputs
   - Impractical for n > 10 cities
   
   Example: 10 cities = 3,628,800 permutations to check
   
   Use Case: Finding true optimal solution for small datasets


2. NEAREST NEIGHBOR ALGORITHM (Greedy Heuristic)
   ----------------------------------------------
   Approach: Start at a city, always move to the closest unvisited city
   
   Algorithm Steps:
   1. Select starting city
   2. Find nearest unvisited city
   3. Move to that city and mark as visited
   4. Repeat until all cities visited
   5. Return to starting city
   
   Time Complexity: O(n²)
   Space Complexity: O(n)
   
   Advantages:
   - Very fast execution
   - Works well for large datasets
   - Easy to implement
   
   Disadvantages:
   - Does not guarantee optimal solution
   - Can produce routes 25-30% longer than optimal
   - Greedy choice may lead to poor overall solution
   
   Use Case: Quick approximation for large TSP instances


3. 2-OPT ALGORITHM (Local Search Optimization)
   -------------------------------------------
   Approach: Improve an existing route by swapping edges
   
   Algorithm Steps:
   1. Start with initial route (from Nearest Neighbor)
   2. Select two edges in the route
   3. Check if swapping them reduces total distance
   4. If yes, perform the swap
   5. Repeat until no improvement possible
   
   Time Complexity: O(n²) per iteration
   Space Complexity: O(n)
   
   Advantages:
   - Significantly improves greedy solutions
   - Still relatively fast
   - Produces near-optimal results
   
   Disadvantages:
   - Requires initial solution
   - May get stuck in local optimum
   - Not guaranteed to find global optimum
   
   Use Case: Improving heuristic solutions for better quality

USER INTERFACE - THREE TAB SYSTEM
================================================================================

TAB 1: PLAYGROUND (City Creation)
----------------------------------
Purpose: Define the city dataset for algorithm testing

Features:
- Click on canvas to manually place cities
- Generate random cities (10 cities at random positions)
- Generate circle pattern (10 cities in circular arrangement)
- Generate clustered cities (12 cities in 3 clusters)
- Clear all cities
- Display city IDs and coordinates

Visual Elements:
- Cities shown as white circles
- City IDs displayed next to each city
- Dark canvas for clear visibility


TAB 2: VISUALIZATION (Algorithm Execution)
-------------------------------------------
Purpose: Watch step-by-step algorithm behavior

Algorithm Selection:
- Brute Force
- Nearest Neighbor
- 2-Opt Improvement

Control Buttons:
- Start: Run algorithm automatically
- Pause: Stop execution to examine current state
- Step: Advance algorithm by one decision
- Reset: Clear and restart algorithm

Visual Feedback:
- Nearest Neighbor: Yellow lines for path, green circle for current city
- Brute Force: Gray lines for current route, green for best route
- 2-Opt: Green lines for route, red circles for edges being swapped

Metrics Displayed:
- Current/Best Distance
- Iterations Count (for Brute Force)
- Elapsed Time (seconds)


TAB 3: COMPARISON (Performance Analysis)
-----------------------------------------
Purpose: Compare all algorithms on same dataset

Features:
- "Run All" button executes all algorithms
- Results table with metrics for each algorithm
- Click algorithm name to display its route
- Color-coded routes for easy identification

Metrics Displayed:

Route Colors:
- Green: Brute Force (optimal solution)

CONCLUSION
================================================================================

This project successfully demonstrates the algorithmic analysis of the 
Travelling Salesman Problem through an interactive visual simulator. 

Key Findings:
1. Algorithm choice significantly impacts both solution quality and 
   computation time

2. Brute Force guarantees optimal solutions but is impractical for 
   real-world applications due to factorial time complexity

3. Nearest Neighbor provides fast approximations suitable for large 
   datasets but may produce suboptimal routes

4. 2-Opt local search effectively improves greedy solutions, offering 
   a practical balance between speed and quality

5. Visual simulation aids in understanding algorithm behavior and 
   decision-making processes

Educational Value:
- Demonstrates NP-Hard problem complexity
- Shows practical algorithm trade-offs
- Provides hands-on experience with optimization techniques
- Illustrates importance of heuristic approaches

The simulator serves as an effective tool for analyzing and comparing 
TSP algorithms, making complex algorithmic concepts accessible through 
interactive visualization.
